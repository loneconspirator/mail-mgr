```typescript
function h(tag: string, attrs: Record<string, string> = {}, ...children: (string | Node)[]): HTMLElement {
  const el = document.createElement(tag);
  for (const [k, v] of Object.entries(attrs)) {
    if (k.startsWith('data-')) el.setAttribute(k, v);
    else (el as any)[k] = v;
  }
  for (const c of children) {
    el.append(typeof c === 'string' ? document.createTextNode(c) : c);
  }
  return el;
}
```

<threat_model>
## Threat Model (ASVS L1)

| Threat | Severity | Mitigation |
|--------|----------|------------|
| XSS via folder names containing HTML | MEDIUM | All folder names rendered via h() helper and document.createTextNode(), never innerHTML with user data |
| Click event propagation (toggle triggers select) | LOW | e.stopPropagation() on toggle click handler |
| Denial of rendering with deeply nested folders | LOW | Max depth guard of 10 levels; real IMAP hierarchies are 5-8 levels |

No high-severity threats. XSS is mitigated by using DOM APIs exclusively for user-generated content (folder names).
</threat_model>

    **Main function `renderFolderPicker(opts)`:**
    - Set container innerHTML to a loading message: `'Loading folders...'` in a span with `color:#888;font-size:0.85rem`
    - Create PickerState with `expanded: new Set()`, `selected: opts.currentValue`, and an `onSelect` that updates `state.selected`, calls `opts.onSelect(path)`, then re-renders via `renderPickerContent()`
    - Fetch `api.folders.list()` and `api.activity.recentFolders()` in parallel with `Promise.all()`
    - On success: auto-expand ancestors of `state.selected` (if set), then call `renderPickerContent(state, folders, recentFolders)`
    - On error: show error div with "Unable to load folders." text and a "Retry" button that calls `renderFolderPicker(opts)` again

    **Cache:** Store fetched folder data in module-level variables with a 60-second TTL:
    ```typescript
    let cachedFolders: FolderNode[] | null = null
    let cachedAt = 0
    const CACHE_TTL = 60_000
    ```
    Before fetching, check `if (cachedFolders && Date.now() - cachedAt < CACHE_TTL)` and skip the API call if cache is fresh.

    **`renderPickerContent(state, folders, recentFolders)` function:**
    - Clear container
    - Create outer div with className `folder-picker`
    - If `state.selected` is non-empty, render a "Selected: <strong>{path}</strong>" bar at top with className `folder-picker-selected`
    - If `recentFolders.length > 0`, render a "Recent" section:
      - Heading div with className `folder-picker-heading`, text "Recent"
      - Section div with className `folder-picker-section`
      - For each recent folder path, render a clickable row (className `tree-node`, add `selected` class if matches `state.selected`) with the folder name (last segment after delimiter `/`). Click calls `state.onSelect(path)`
    - Render "All Folders" section:
      - Heading div with className `folder-picker-heading`, text "All Folders"
      - Section div with className `folder-picker-section`
      - For each root FolderNode, call `renderTreeNode(node, 0, state)` and append

    **`renderTreeNode(node, depth, state)` function:**
    - Guard: if `depth > 10` return an empty div (safety)
    - Create row div with className `tree-node`. Add class `selected` if `node.path === state.selected`
    - Set `paddingLeft` to `${depth * 16}px` via style attribute
    - If `node.children.length > 0`: create toggle span (className `tree-toggle`) with text `\u25BE` (down triangle) if expanded, `\u25B8` (right triangle) if not. Add click listener with `e.stopPropagation()` that toggles `state.expanded` set and re-renders
    - If no children: create spacer span (className `tree-toggle tree-leaf`)
    - Create label span (className `tree-label`) with `node.name` as text content (use createTextNode, NOT innerHTML)
    - Add click listener on row that calls `state.onSelect(node.path)`
    - Append toggle and label to row
    - If expanded and has children: for each child, recursively call `renderTreeNode(child, depth + 1, state)` and append after row inside a wrapper, OR return a fragment/array. Simpler approach: create a container div, append row, then if expanded append child nodes.
    - Return the container

    **`expandPathTo(folders, targetPath, expanded)` helper:**
    - Recursively walk the tree. For each node, if `targetPath` starts with `node.path + node.delimiter`, add `node.path` to `expanded` set and recurse into children.
