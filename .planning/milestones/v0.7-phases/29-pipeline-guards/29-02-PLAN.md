Add sentinel message guards to all 5 message processors so they skip sentinel messages without processing them.

Purpose: Prevents sentinel tracking messages from being treated as real mail -- action folders won't create rules from them, monitor won't evaluate them, sweeper won't archive them, batch won't file them, tracker won't snapshot them.

Output: Each processor has an early-exit guard per D-03/D-04 pattern, with test coverage proving sentinels are skipped.
</objective>
