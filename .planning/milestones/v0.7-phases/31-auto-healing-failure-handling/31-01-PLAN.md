<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Scanner report -> Healer | ScanReport data drives config mutations; report comes from trusted internal scanner |
| Healer -> Config file | Healer writes config.yml via saveConfig; must not corrupt config |
| Healer -> INBOX | Healer APPENDs notification messages; must not inject malicious content |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-31-01 | Tampering | Config mutation | mitigate | saveConfig() validates via configSchema.safeParse() before writing -- invalid mutations throw before persisting |
| T-31-02 | Denial of Service | Notification spam | mitigate | Dedup tracking via activityLog.setState prevents repeated INBOX notifications per D-06 |
| T-31-03 | Information Disclosure | Notification content | accept | Notification only contains folder paths and rule names (no credentials/PII); delivered to user's own INBOX |
| T-31-04 | Tampering | Race condition | accept | Node.js single-threaded event loop prevents concurrent config mutation (A1 from RESEARCH); low risk |
</threat_model>
