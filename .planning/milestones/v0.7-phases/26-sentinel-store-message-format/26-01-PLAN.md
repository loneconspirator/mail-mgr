GREEN phase: Create `src/sentinel/format.ts` with:

1. Export a `FolderPurpose` union type: `'rule-target' | 'action-folder' | 'review' | 'sweep-target'`

2. Export interface `BuildSentinelOpts`:
   - `folderPath: string`
   - `folderPurpose: FolderPurpose`
   - `bodyText?: string` (optional -- if omitted, use purposeBody helper)

3. Export interface `SentinelMessage`:
   - `raw: string` (RFC 2822 message for IMAP APPEND)
   - `messageId: string` (full `<uuid@mail-manager.sentinel>` form with angle brackets)
   - `flags: string[]` (always `['\\Seen']`)

4. Export function `buildSentinelMessage(opts: BuildSentinelOpts): SentinelMessage`:
   - Guard: if `opts.folderPath.toUpperCase() === 'INBOX'` throw `new Error('Cannot create sentinel for INBOX')`
   - Guard: if `opts.folderPath` contains `\r` or `\n`, throw `new Error('Folder path contains invalid characters')`
   - Generate UUID via `randomUUID()` from `node:crypto`
   - Build `messageId` as `<${uuid}@mail-manager.sentinel>` (per D-06)
   - Build `date` via `new Date().toUTCString()` (RFC 2822-compatible)
   - Construct headers array joined with `\r\n`:
     - `Message-ID: ${messageId}`
     - `Date: ${date}`
     - `From: mail-manager@localhost` (per D-02)
     - `To: mail-manager@localhost`
     - `Subject: [Mail Manager] Sentinel: ${opts.folderPath}` (per D-01)
     - `X-Mail-Mgr-Sentinel: ${messageId}` (per D-04)
     - `MIME-Version: 1.0`
     - `Content-Type: text/plain; charset=utf-8`
   - Join: `raw = headers.join('\r\n') + '\r\n\r\n' + bodyText`
   - Body text: use `opts.bodyText` if provided, otherwise call `purposeBody(opts.folderPath, opts.folderPurpose)`
   - Return `{ raw, messageId, flags: ['\\Seen'] }`

5. Export function `purposeBody(folderPath: string, purpose: FolderPurpose): string` that returns descriptive text per D-03:
   - `'rule-target'`: "This message is a Mail Manager sentinel. It tracks the folder '{folderPath}' so that folder renames can be automatically detected and all references updated. Please do not delete this message."
   - `'action-folder'`: "This message is a Mail Manager sentinel. It tracks the action folder '{folderPath}'. Moving messages into this folder triggers automatic processing. If this folder is renamed, Mail Manager will detect the change and update its configuration. Please do not delete this message."
   - `'review'`: "This message is a Mail Manager sentinel. It tracks the review folder '{folderPath}'. Messages are placed here for periodic review before archiving. Please do not delete this message."
   - `'sweep-target'`: "This message is a Mail Manager sentinel. It tracks the sweep target folder '{folderPath}'. Reviewed messages are swept here after the review period. Please do not delete this message."

REFACTOR phase: Verify all tests pass. Clean up if needed.
