    Add `reconcileSentinels` to `src/sentinel/lifecycle.ts`.

    Function signature per D-06, D-07:
    ```typescript
    interface Logger {
      info(obj: Record<string, unknown>, msg: string): void;
      warn(obj: Record<string, unknown>, msg: string): void;
    }

    export async function reconcileSentinels(
      tracked: Map<string, FolderPurpose>,
      store: SentinelStore,
      client: ImapClient,
      logger: Logger,
    ): Promise<{ planted: number; removed: number; errors: number }>
    ```

    Implementation:
    1. Call `store.getAll()` to get existing sentinels
    2. Build `existingFolders` as `new Set(existing.map(s => s.folderPath))`
    3. Plant missing: for each `[folder, purpose]` in `tracked`, if `!existingFolders.has(folder)`, call `appendSentinel(client, folder, purpose, store)` inside try/catch. Increment `planted` on success, `errors` on failure. Log warning on failure with `{ err, folder }`.
    4. Remove orphaned: for each sentinel in `existing`, if `!tracked.has(sentinel.folderPath)`:
       - Call `findSentinel(client, sentinel.folderPath, sentinel.messageId)` to get UID
       - If UID found: call `deleteSentinel(client, sentinel.folderPath, uid, store, sentinel.messageId)`
       - If UID undefined: call `store.deleteByMessageId(sentinel.messageId)` (store-only cleanup)
       - Wrap in try/catch, increment `errors` on failure, `removed` on success
    5. Return `{ planted, removed, errors }`

    Add tests using vi.fn() mocks for ImapClient, SentinelStore, and the imap-ops functions. Mock `appendSentinel`, `findSentinel`, `deleteSentinel` at the module level using `vi.mock('../../../src/sentinel/imap-ops.js')`.
