<objective>
When approving a proposed rule (approve, approve-as-review, or modify), assign `order` to be one more than the current max order so the new rule sorts to the bottom of the rules list instead of the top.

Purpose: Currently all approved proposals get `order: 0`, which puts them at the top of the rule list and can shadow existing higher-order rules.
Output: Modified proposed-rules route and frontend to compute next order dynamically.
</objective>

1. In `src/config/repository.ts`, add a `nextOrder()` method to ConfigRepository:
   ```typescript
   nextOrder(): number {
     const rules = this.getRules();
     if (rules.length === 0) return 0;
     return Math.max(...rules.map(r => r.order)) + 1;
   }
   ```
