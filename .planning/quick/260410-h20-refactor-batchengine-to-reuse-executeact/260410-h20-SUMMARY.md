# Quick Task 260410-h20: Refactor BatchEngine to reuse executeAction and processSweepMessage

Eliminated code duplication in BatchEngine by reusing executeAction for INBOX mode and extracting a shared processSweepMessage for Review mode sweep processing.

## Changes

### src/actions/index.ts
- Added optional `sourceFolder` field to `ActionContext` interface
- Updated `executeMove` to accept full `ActionContext` and pass `sourceFolder` through to `client.moveMessage`
- Backward-compatible: callers that don't set `sourceFolder` get `undefined`, which the IMAP client defaults to `'INBOX'`

### src/sweep/index.ts
- Extracted `processSweepMessage` function from `ReviewSweeper.runSweep()` inline loop body
- New exported interfaces: `ProcessSweepMessageDeps`, `ProcessSweepMessageResult`
- `ReviewSweeper.runSweep()` now delegates to `processSweepMessage` instead of inline try/catch
- Supports both `'sweep'` and `'batch'` source labels via `ProcessSweepMessageDeps.source`

### src/batch/index.ts
- INBOX mode `execute()` now calls `executeAction` with `ActionContext` (including `sourceFolder`) instead of manual `resolveDestination` + `moveMessage`
- Review mode `execute()` now calls `processSweepMessage` instead of inline sweep logic (manual `resolveSweepDestination` + `moveMessage` + `ActionResult` construction)
- Generic mode retains existing manual handling (behavioral difference in review action routing)
- `resolveDestination` private method preserved for dry-run and generic mode usage

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Batch test mock needed processSweepMessage**
- **Found during:** Task 2
- **Issue:** The batch test file mocks the entire sweep module with `vi.mock()`, providing only `isEligibleForSweep` and `resolveSweepDestination`. After refactoring batch to import `processSweepMessage`, the mock module returned `undefined` for it, causing 2 Review mode tests to fail silently.
- **Fix:** Added `processSweepMessage` to the mock factory and updated 2 Review mode test assertions to verify `processSweepMessage` was called with correct deps rather than checking `moveMessage` directly.
- **Files modified:** test/unit/batch/engine.test.ts
- **Commit:** f412d1f

**2. [Rule 3 - Blocking] Action tests needed sourceFolder parameter update**
- **Found during:** Task 1
- **Issue:** `executeMove` now passes `ctx.sourceFolder` (undefined when not set) as a third arg to `moveMessage`. Existing action tests used `toHaveBeenCalledWith(uid, folder)` with 2 args, which fails strict argument matching against 3 args.
- **Fix:** Updated 4 action test assertions to expect `undefined` as the third argument.
- **Files modified:** test/unit/actions/actions.test.ts
- **Commit:** 49a9ac0

**3. [Rule 2 - Missing] Generic mode behavioral difference preserved**
- **Found during:** Task 2
- **Issue:** `executeAction` routes review actions unconditionally to `ctx.reviewFolder`, but generic batch mode expects review rules with an explicit folder to route to that folder (e.g., 'Newsletters' not 'Review'). Using `executeAction` for generic mode would break test expectations.
- **Fix:** Only applied `executeAction` delegation for INBOX mode. Generic mode retains existing manual `resolveDestination` + `moveMessage` handling.
- **Files modified:** src/batch/index.ts
- **Commit:** f412d1f
