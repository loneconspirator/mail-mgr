**src/sweep/index.ts — Extract processSweepMessage:**

Extract a new exported async function `processSweepMessage` from the body of `ReviewSweeper.runSweep()`'s for-loop (lines 185-224). The function processes a single ReviewMessage through the sweep pipeline:

```typescript
export interface ProcessSweepMessageDeps {
  client: ImapClient;
  activityLog: ActivityLog;
  rules: Rule[];
  defaultArchiveFolder: string;
  trashFolder: string;
  sourceFolder: string;
  source: 'sweep' | 'batch';
  logger: pino.Logger;
}

export interface ProcessSweepMessageResult {
  action: 'moved' | 'error';
  destination: string;
}

export async function processSweepMessage(
  msg: ReviewMessage,
  deps: ProcessSweepMessageDeps,
): Promise<ProcessSweepMessageResult>
```

The function body should:
1. Call `resolveSweepDestination(msg, deps.rules, deps.defaultArchiveFolder)` to get destination and matchedRule.
2. Resolve folder: `dest.type === 'delete' ? deps.trashFolder : dest.folder`.
3. Convert to EmailMessage via `reviewMessageToEmailMessage(msg)`.
4. Try `deps.client.moveMessage(msg.uid, folder, deps.sourceFolder)`.
5. On success: build `ActionResult` with `success: true`, call `deps.activityLog.logActivity(result, emailMsg, matchedRule, deps.source)`, return `{ action: 'moved', destination: folder }`.
6. On error: log error via `deps.logger.error(...)`, build `ActionResult` with `success: false` and error string, call `deps.activityLog.logActivity(...)`, return `{ action: 'error', destination: folder }`.
7. Use `matchedRule?.name ?? ''` for the rule field in ActionResult (matching current sweep behavior, NOT `matchedRule?.id`).

Then refactor `ReviewSweeper.runSweep()` to call `processSweepMessage` in its loop instead of the inline code. The loop becomes:
