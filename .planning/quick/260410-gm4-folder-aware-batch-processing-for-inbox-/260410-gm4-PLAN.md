<task type="auto" tdd="true">
  <name>Task 1: Add folder-aware processing modes to BatchEngine</name>
  <files>src/batch/index.ts, src/index.ts</files>
  <behavior>
    - INBOX mode (sourceFolder === 'INBOX'):
      - review actions route to reviewFolder (not to action.folder or Skip)
      - skip actions leave message in place (skipped count increments)
      - move actions work as before
      - delete actions work as before
      - no-match messages are skipped (left in INBOX, like monitor)
    - Review mode (sourceFolder === reviewFolder):
      - messages not eligible for sweep (by age) are skipped entirely
      - eligible messages use resolveSweepDestination to find target folder
      - delete destinations route to trashFolder
      - move destinations route to resolved folder
      - no-match messages route to defaultArchiveFolder (via resolveSweepDestination)
    - Generic mode (any other folder):
      - current behavior preserved exactly (review actions resolve to action.folder or Skip)
  </behavior>
  <action>
1. Update `BatchDeps` interface to add optional `reviewFolder: string` and `reviewConfig: ReviewConfig` fields. These are optional so existing callers don't break, but required for INBOX/Review mode to work.

2. Add a private method `private getProcessingMode(sourceFolder: string): 'inbox' | 'review' | 'generic'` that returns:
   - `'inbox'` when sourceFolder is `'INBOX'` (case-sensitive, standard IMAP)
   - `'review'` when sourceFolder equals `this.deps.reviewFolder`
   - `'generic'` otherwise

3. Update `resolveDestination(matched: Rule)` to accept sourceFolder and branch by mode:
   - **inbox mode**: For `review` action type, return `this.deps.reviewFolder` (the review folder itself). For `skip`, return `'Skip'`. For `move` and `delete`, same as current.
   - **review mode**: Should NOT use resolveDestination at all (handled separately via resolveSweepDestination).
   - **generic mode**: Current logic unchanged.

4. In `dryRun()`, add review-mode handling:
   - Import `isEligibleForSweep` and `resolveSweepDestination` from `'../sweep/index.js'`.
   - When mode is `'review'`: for each message, first check `isEligibleForSweep(raw, this.deps.reviewConfig!.sweep, new Date())`. If not eligible, group as `{ destination: 'Not yet eligible', action: 'skip' }`. If eligible, call `resolveSweepDestination(raw, this.deps.rules, this.deps.reviewConfig!.defaultArchiveFolder)` and use that result for grouping.
   - When mode is `'inbox'`: use the updated resolveDestination.
   - When mode is `'generic'`: current logic.

5. In `execute()`, add review-mode handling:
   - When mode is `'review'`: for each message raw, check `isEligibleForSweep`. If not eligible, increment skipped and continue. If eligible, call `resolveSweepDestination` to get destination. If destination is delete, move to trashFolder. Otherwise move to destination folder. Log activity with source `'batch'`.
   - When mode is `'inbox'`: use updated resolveDestination. Review actions move to reviewFolder. Skip actions skip.
   - When mode is `'generic'`: current logic unchanged.

6. Update `src/index.ts`: Pass `reviewFolder: config.review.folder` and `reviewConfig: config.review` to both BatchEngine constructors (initial and hot-reload).
  </action>

  </verify>
