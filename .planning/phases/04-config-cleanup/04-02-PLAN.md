Per D-06: investigate whether the stale sweeper reference is a real timing gap or already resolved by the getter pattern.

**Investigation findings from RESEARCH.md (Open Question 1):**
The `getSweeper: () => sweeper` getter captures the `let sweeper` binding, not the value. When `onReviewConfigChange` runs:
1. `sweeper.stop()` (line 70) — old sweeper stopped
2. Lines 71-72 — async IMAP call to get trash folder
3. `sweeper = new ReviewSweeper(...)` (line 73) — new sweeper assigned
4. `sweeper.start()` (line 81) — new sweeper started

**The bug is REAL:** Between step 1 (stop) and step 3 (reassignment), any route calling `getSweeper()` gets the STOPPED old sweeper. The async gap at step 2 (`getSpecialUseFolder`) makes this window non-trivial.

**Fix:** Add a guard in `onReviewConfigChange` to set sweeper to undefined before the async gap, so routes know the sweeper is rebuilding:

In `src/index.ts`, modify the `onReviewConfigChange` handler (line 68-82):
```typescript
configRepo.onReviewConfigChange(async () => {
  const updatedConfig = configRepo.getConfig();
  sweeper.stop();
  sweeper = undefined as any;  // Signal "rebuilding" to getSweeper() callers
  const reviewTrash = await imapClient.getSpecialUseFolder('\\Trash')
    ?? updatedConfig.review.trashFolder;
  sweeper = new ReviewSweeper({
    client: imapClient,
    activityLog,
    rules: updatedConfig.rules,
    reviewConfig: updatedConfig.review,
    trashFolder: reviewTrash,
    logger,
  });
  sweeper.start();
});
```

Also update the `ServerDeps.getSweeper` type usage — it already returns `ReviewSweeper | undefined`, so callers already need to handle undefined. Verify the review status route handles `getSweeper()` returning undefined gracefully. Check `src/web/routes/review.ts` or wherever sweeper is used in routes.

**Add a test** in `test/unit/config/repository.test.ts` or a new integration test that verifies: after calling `updateReviewConfig`, the onReviewConfigChange listener fires and the getter returns the new instance after the callback completes.
