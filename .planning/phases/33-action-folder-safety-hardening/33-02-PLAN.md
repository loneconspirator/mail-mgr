Output: poller.ts with sentinel-aware skip logic and updated/new tests.

Modify `src/action-folders/poller.ts` scanAll method. Replace the current skip logic at lines 37-40:

**Current code:**
```typescript
const { messages } = await this.deps.client.status(path);
if (messages === 0) continue;

this.deps.logger.info({ folder: path, count: messages }, 'Processing action folder');
const rawMessages = await this.deps.client.fetchAllMessages(path);
```

**Replace with (per D-01, D-02, D-03):**
```typescript
const { messages } = await this.deps.client.status(path);
if (messages === 0) {
  this.deps.logger.debug({ folder: path }, 'Action folder empty (sentinel missing), skipping');
  continue;
}
if (messages === 1) {
  this.deps.logger.debug({ folder: path }, 'Action folder has only sentinel, skipping fetch');
  continue;
}

this.deps.logger.info({ folder: path, count: messages }, 'Processing action folder');
const rawMessages = await this.deps.client.fetchAllMessages(path);
```

Use `debug` level for both skip logs (per discretion -- these fire every 15 seconds per folder, `info` would drown meaningful logs).

The rest of the scanAll method (sentinel counting in the processing loop, FOLD-02 retry logic) remains unchanged. The sentinel skip is upstream of all that code and only fires when messages <= 1.
