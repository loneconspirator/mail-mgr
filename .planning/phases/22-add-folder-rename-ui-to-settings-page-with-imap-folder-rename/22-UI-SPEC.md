## Interaction States

### Folder Selection

| State | Visual |
|-------|--------|
| No folder selected | Tree picker visible, rename field hidden. Empty state message shown below picker. |
| Folder selected | Selected folder highlighted with `background: #e8f0fe` (light blue). Rename field appears below tree. |
| Non-renamable selected (INBOX or Actions/*) | Selection visually muted (`opacity: 0.5`), rename field shows disabled message instead of input. |

### Rename Operation

| State | Visual |
|-------|--------|
| Idle | Input shows current leaf folder name, "Rename Folder" button enabled |
| Submitting | Button disabled, shows spinner + "Renaming..." text, input disabled |
| Success | Toast "Folder renamed" shown, tree refreshes, selection cleared |
| Error (IMAP failure) | Toast with error message (red), input re-enabled, tree refreshes |
| Name collision | Inline error text below input: "A folder named {name} already exists here" (no IMAP call) |

### Special-Use Folder Warning

| Trigger | Visual |
|---------|--------|
| User selects Sent, Drafts, Trash, Junk, or Archive | Warning banner appears above rename input: yellow background (#fef9c3), dark text (#854d0e), text: "This is a special-use folder. Renaming may affect your mail client." |

## Layout

```
+------------------------------------------+
| settings-card                            |
|                                          |
|  <h2>Folder Management</h2>             |
|                                          |
|  [Folder Tree Picker]                    |
|    INBOX                                 |
|    > Projects                            |
|      > Active     <-- selected           |
|    > Archive                             |
|                                          |
|  ┌─ rename section (visible when ────┐   |
|  │  selected & renamable)            │   |
|  │                                   │   |
|  │  [warning if special-use]         │   |
|  │                                   │   |
|  │  Label: "New name"               │   |
|  │  [  Active__________________ ]    │   |
|  │  [collision error if any]         │   |
|  │                                   │   |
|  │  [ Cancel ]  [ Rename Folder ]    │   |
|  └───────────────────────────────────┘   |
|                                          |
+------------------------------------------+
```

Card position: After the existing "Sweep Settings" card, before any future cards. This keeps folder operations grouped near other folder-related settings.

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Keyboard nav | Tree picker already handles arrow keys. Rename input receives focus on folder select. |
| Button disabled state | `disabled` attribute + `opacity: 0.5` + `pointer-events: none` (existing `.btn:disabled`) |
| Error announcement | Collision error uses `role="alert"` for screen reader announcement |
| Warning announcement | Special-use warning uses `role="status"` |
