Three changes across three files:

**1. api.ts — Add cursor toggle methods to config namespace (per D-04)**

Add two methods to the `api.config` object after `updateReview`:

```typescript
getCursor: () => request<{ enabled: boolean }>('/api/settings/cursor'),
setCursor: (enabled: boolean) => request<void>('/api/settings/cursor', { method: 'PUT', body: JSON.stringify({ enabled }) }),
```

Return type `{ enabled: boolean }` for getCursor matches the existing backend response shape. setCursor returns void (PUT with no meaningful response body).

**2. app.ts — Fix no-match filter (per D-01, D-02, D-03)**

In `renderBatchPreview` function (around line 600), replace the filter logic:

OLD (line 600):
```typescript
const matchGroups = groups.filter(g => g.action !== 'skip' || g.destination !== '');
const noMatchGroup = groups.find(g => g.action === 'skip' && g.destination === '');
```

NEW:
```typescript
const matchGroups = groups.filter(g => g.action !== 'no-match');
const noMatchGroup = groups.find(g => g.action === 'no-match');
```

The backend BatchEngine already returns `action='no-match'` for unmatched messages. The old filter was checking for `action === 'skip' && destination === ''` which never matched the actual data.
