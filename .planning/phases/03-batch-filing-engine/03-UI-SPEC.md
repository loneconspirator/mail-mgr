## Component Inventory — Phase 3

Phase 3 needs UI for a multi-step batch workflow: select source folder, run dry-run, review preview, execute or cancel, view results. This is a new top-level page ("Batch") added to the navigation.

### New Components

#### 1. Batch Page (top-level view)

Container for the batch filing workflow. Rendered when user clicks the "Batch" nav button. Manages the step progression.

- **Location:** new function `renderBatch()` in `app.ts` (follows `renderRules()`, `renderActivity()`, `renderSettings()` pattern)
- **Nav integration:** Add a fourth nav button `<button class="nav-btn" data-page="batch">Batch</button>` to `index.html`

#### 2. Source Folder Selector Card

A `settings-card` containing the Phase 2 tree picker for selecting the batch source folder, plus a "Start Dry Run" button.

| Element | Specification |
|---------|---------------|
| Card | `.settings-card` (existing class) |
| Heading | `<h2>Batch Filing</h2>` at 18px/600 |
| Instructions | Body text: "Select a source folder to apply all rules against its messages." at 14px/400, color #444 |
| Folder picker | Reuse `renderFolderPicker()` from `folder-picker.ts` (D-04) |
| CTA button | `.btn.btn-primary`, label: "Preview Dry Run", disabled until folder selected |
| Disabled state | Button uses `opacity: 0.5; cursor: not-allowed` when no folder selected |

#### 3. Dry-Run Progress Indicator

Shown while the dry-run evaluation runs server-side. Replaces the selector card content.

| Element | Specification |
|---------|---------------|
| Container | `.settings-card` |
| Spinner text | "Evaluating rules against {folderName}..." at 14px/400, color #888, centered |
| Animation | CSS `@keyframes` pulse on the text (opacity 0.4 to 1.0, 1.5s infinite) |

#### 4. Dry-Run Results Panel (Preview)

Displays grouped results from dry-run. Each destination folder is a collapsible group showing message count and expandable message list (D-05, D-06).

| Element | Specification |
|---------|---------------|
| Container | `.settings-card` |
| Heading | `<h2>Dry Run Preview</h2>` |
| Summary line | "{totalMatched} of {totalMessages} messages matched" at 14px/400 |
| Group row | Flex row: destination folder name (14px/600) + count badge (13px/400, muted) + expand toggle (Unicode triangle) |
| Group row hover | `background: #f0f4ff` (matches tree-node hover) |
| Expanded message list | Table inside group: columns From, Subject, Rule. Font 13px/400. Max-height 200px with overflow-y scroll |
| No-match group | Displayed last, destination label: "No match (stay in folder)", color #888 |
| Action bar | Flex row, justify-content: space-between. Left: "Back" button (`.btn`). Right: "Run Batch" button (`.btn.btn-primary`) |
| Run Batch button | Label: "Run Batch" — transitions from preview to execution (D-07) |
| Back button | Label: "Back" — returns to source folder selector |

#### 5. Batch Execution Status Card

Shown during active batch execution. Displays progress and cancel button (D-08).

| Element | Specification |
|---------|---------------|
| Container | `.settings-card` |
| Heading | `<h2>Batch Running</h2>` |
| Progress text | "{processed} of {totalMessages} messages processed" at 14px/400 |
| Progress bar | Height 8px, border-radius 4px, background #eee, fill #2563eb, width proportional to processed/total |
| Counts row | Three inline stats: "Moved: {n}" / "Skipped: {n}" / "Errors: {n}" at 13px/400, spaced with 16px gaps |
| Error count | Color #dc2626 when errors > 0 |
| Cancel button | `.btn.btn-danger`, label: "Cancel Batch" (D-08) |
| Poll interval | Frontend polls `GET /api/batch/status` every 2 seconds while status is `executing` |

#### 6. Batch Results Summary Card

Shown after batch completes, cancels, or errors (D-09).

| Element | Specification |
|---------|---------------|
| Container | `.settings-card` |
| Heading | `<h2>Batch Complete</h2>` or `<h2>Batch Cancelled</h2>` or `<h2>Batch Error</h2>` |
| Status badge | `.status-badge` with connected/disconnected/connecting class reused for complete/error/cancelled |
| Stats grid | Reuse `.review-stats` layout (3-column grid): Moved / Skipped / Errors, each with `.stat-value` (24px/700) and `.stat-label` (13px/600 uppercase) |
| Remaining count | Shown only on cancellation: "Remaining: {n} messages not processed" at 14px/400, color #888 |
| Action | "New Batch" button (`.btn.btn-primary`) — resets to source folder selector |

#### 7. Batch Activity Badge

Activity log entries with `source='batch'` display a badge, mirroring the existing `badge-sweep` pattern.

| Element | Specification |
|---------|---------------|
| Badge | `.badge-batch`, styled identically to `.badge-sweep` but with distinct color |
| Colors | background: #fef3c7, color: #92400e (amber, distinct from sweep's indigo) |
| Content | `[batch]` text |
| Placement | Before rule name in activity table, same as sweep badge |

## Interaction States

### Batch Workflow State Machine (UI perspective)

```
[idle] → select folder → [folder-selected] → click "Preview Dry Run"
  → [dry-running] → dry-run completes → [previewing]
  → expand/collapse groups → [previewing]
  → click "Run Batch" → [executing]
  → click "Cancel Batch" → [cancelling] → chunk completes → [cancelled]
  → batch completes → [completed]
  → click "Back" (from previewing) → [idle]
  → click "New Batch" (from completed/cancelled) → [idle]
```

### Disabled States

| State | What is Disabled | How |
|-------|-----------------|-----|
| No folder selected | "Preview Dry Run" button | `opacity: 0.5; cursor: not-allowed; pointer-events: none` |
| Batch active (any status except idle) | All nav buttons except Batch | Do NOT disable nav — user can navigate away and return (D-13) |
| Batch executing | "Run Batch" button (already replaced by execution view) | Button not rendered |

### Loading States

| State | Display |
|-------|---------|
| Folder picker loading | "Loading folders..." text (existing pattern) |
| Dry-run running | Pulsing "Evaluating rules against {folder}..." text |
| Batch executing | Progress bar + live counts, polled every 2s |
| Page load during active batch | Show execution status card immediately (poll on mount) |

### Error States

| Error | Display |
|-------|---------|
| Dry-run API failure | Toast error + return to folder selector |
| Batch start failure | Toast error + stay on preview |
| Batch already running (D-12) | "Preview Dry Run" button disabled + info text: "A batch is already running." |
| IMAP disconnected | Toast: "IMAP connection lost. Batch will resume when reconnected." |

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | "Preview Dry Run" (step 1), "Run Batch" (step 2) |
| Empty state heading | "Batch Filing" |
| Empty state body | "Select a source folder to apply all rules against its messages." |
| Error state (dry-run fail) | "Dry run failed: {error message}. Check your IMAP connection and try again." |
| Error state (batch fail) | "Batch error: {error message}. Already-moved messages remain in their destinations." |
| Error state (already running) | "A batch is already running. Wait for it to complete or cancel it first." |
| Destructive: Cancel Batch | Button label: "Cancel Batch". No confirmation dialog — cancellation is safe (stops after current chunk, D-08). Post-cancel copy: "Batch cancelled. {moved} messages moved, {remaining} messages remain in {folder}." |
| Destructive: Run Batch | Button label: "Run Batch". No confirmation dialog — dry-run preview IS the confirmation step (D-07). |
| Batch complete | "Batch complete. {moved} messages filed to {n} folders." |
| Batch cancelled | "Batch cancelled. {moved} messages moved, {remaining} not processed." |
| No matches | "No match (stay in folder)" — group label for unmatched messages in dry-run preview |
| Nav button | "Batch" |
