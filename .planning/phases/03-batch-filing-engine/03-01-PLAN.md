## Threat Model

| Threat | Severity | Mitigation |
|--------|----------|------------|
| Batch engine processes messages from wrong folder | medium | sourceFolder parameter validated as non-empty string; test verifies fetchAllMessages called with correct folder |
| Cancel flag race condition between chunks | low | cancelRequested is a simple boolean checked synchronously between chunks; no race possible in single-threaded Node.js |
| Memory exhaustion on very large folders | low | Accepted for v1 per RESEARCH.md — ReviewMessage envelopes are ~200 bytes each, 10K messages = ~2MB |
| Batch runs while another batch is active | medium | running guard with thrown Error; test verifies double-start throws |

No high/critical threats identified. All medium threats have mitigations.

    ```typescript
    export type BatchStatus = 'idle' | 'dry-running' | 'previewing' | 'executing' | 'completed' | 'cancelled' | 'error'

    export interface BatchDeps {
      client: ImapClient
      activityLog: ActivityLog
      rules: Rule[]
      reviewFolder: string
      trashFolder: string
      logger?: pino.Logger
    }

    export interface BatchState {
      status: BatchStatus
      sourceFolder: string | null
      totalMessages: number
      processed: number
      moved: number
      skipped: number
      errors: number
      cancelled: boolean
      dryRunResults: DryRunGroup[] | null
      completedAt: string | null
    }

    export interface DryRunGroup {
      destination: string   // folder path, 'no-match', or 'skip'
      action: string        // 'move' | 'delete' | 'skip' | 'review' | 'no-match'
      count: number
      messages: DryRunMessage[]
    }

    export interface DryRunMessage {
      uid: number
      from: string
      subject: string
      date: string
      ruleName: string
    }

    export interface BatchResult {
      status: 'completed' | 'cancelled' | 'error'
      totalMessages: number
      processed: number
      moved: number
      skipped: number
      errors: number
      completedAt: string
    }
    ```

    **BatchEngine class:**
    - Constructor takes `BatchDeps`, stores deps as private fields. Create pino logger with name 'batch' if not provided.
    - Private fields: `cancelRequested = false`, `running = false`, state initialized to idle defaults.
    - `CHUNK_SIZE = 25` as module-level constant.

    **dryRun(sourceFolder: string): Promise<DryRunGroup[]>**
    - Guard: if `this.running` throw `new Error('Batch already running')`
    - Set `running = true`, status = 'dry-running', sourceFolder on state
    - Fetch all messages via `this.deps.client.fetchAllMessages(sourceFolder)`
    - Set `state.totalMessages = messages.length`
    - For each message: convert via `reviewMessageToEmailMessage(raw)`, call `evaluateRules(this.deps.rules, msg)`
    - Group results by destination key: for matched rules, key = `${rule.action.type}:${resolveDestination(rule)}` where resolveDestination returns the folder path for move actions, reviewFolder for review actions, trashFolder for delete actions, 'skip' for skip actions. For unmatched, key = 'no-match'.
    - Each group: `{ destination: folderPath or 'No match' or 'Skip', action: actionType or 'no-match', count, messages: [{ uid, from: msg.from.address, subject: msg.subject, date: msg.date.toISOString(), ruleName: rule?.name ?? '' }] }`
    - Set `state.dryRunResults = groups`, `state.status = 'previewing'`
    - Return groups array. In `finally`: set `running = false`.
    - On error: set `state.status = 'error'`, re-throw.

    **execute(sourceFolder: string): Promise<BatchResult>**
    - Guard: if `this.running` throw `new Error('Batch already running')`
    - Set `running = true`, `cancelRequested = false`, reset counters to 0, status = 'executing', sourceFolder on state
    - Fetch all messages via `this.deps.client.fetchAllMessages(sourceFolder)`
    - Set `state.totalMessages = messages.length`
    - Build ActionContext: `{ client: this.deps.client, reviewFolder: this.deps.reviewFolder, trashFolder: this.deps.trashFolder }`
    - Process in chunks of CHUNK_SIZE:
      ```
      for (let i = 0; i < messages.length; i += CHUNK_SIZE) {
        if (this.cancelRequested) { this.state.status = 'cancelled'; break; }
        const chunk = messages.slice(i, i + CHUNK_SIZE)
        for (const raw of chunk) {
          const msg = reviewMessageToEmailMessage(raw)
          const matched = evaluateRules(this.deps.rules, msg)
          if (!matched) { this.state.skipped++; this.state.processed++; continue; }
          try {
            const result = await executeAction(ctx, msg, matched)
            this.deps.activityLog.logActivity(result, msg, matched, 'batch')
            if (result.success) this.state.moved++
            else this.state.errors++
          } catch (err) {
            this.state.errors++
            this.logger.error({ uid: raw.uid, error: err instanceof Error ? err.message : String(err) }, 'Batch message failed')
          }
          this.state.processed++
        }
        // Yield between chunks
        if (i + CHUNK_SIZE < messages.length && !this.cancelRequested) {
          await new Promise<void>((resolve) => setImmediate(resolve))
        }
      }
      ```
    - If status still 'executing' (not cancelled), set status = 'completed'
    - In `finally`: set `running = false`, `state.completedAt = new Date().toISOString()`
    - Return `this.buildResult()` which creates a `BatchResult` from current state.

    **cancel(): void**
    - If `this.running`, set `this.cancelRequested = true`

    **getState(): BatchState**
    - Return `{ ...this.state, dryRunResults: this.state.dryRunResults ? [...this.state.dryRunResults] : null }`

    **updateRules(rules: Rule[]): void**
    - `this.deps.rules = rules`

    **Private resetState(): void**
    - Reset state to idle defaults: all counters 0, sourceFolder null, dryRunResults null, completedAt null, cancelled false, status 'idle'

    **Private buildResult(): BatchResult**
    - Return `{ status: this.state.status as 'completed' | 'cancelled' | 'error', totalMessages, processed, moved, skipped, errors, completedAt: this.state.completedAt! }`
